如何在VS中使用git管理源代码

作为世界第一IDE,VS的优越性不需要我多说了8，同样，作为项目管理与代码版本控制的卓越工具，git也深受欢迎。本文就简单介绍如何在VS中使用git 插件对源代码进行管理。

准备条件

git 安装好， 不多说

vs 安装好，本文用vs2019

代码托管用github, 有设置好github， 包括ssh等一系列操作

在vs中使用git插件(已经集成)，在菜单栏中选择“工具”，再进行下列操作。

一般使用给自己的项目加入源代码管理分为几种情况，一种是加入别人的项目，也就是从一个已经有的项目分支克隆到本地；另一种是用本地新建的项目初始化推送到远程仓库并进行后续管理，下面一一进行介绍。

克隆远程仓库已有分支

此情况为：当作为一个新成员参与别人已经建好的项目中，此时远程已经有有一些分支了。

1.克隆(clone)

从远程下载代码步骤如下：打开VS，点击克隆或签出代码

按照下图填好远程仓库URI和本地URI即可

这个时候，就会发现在团队资源管理器中显式已成功克隆，在下面的解决方案栏中点击打开，找到对应的sln问价即可打开远程分支的项目文件，在解决方案管理器 中可以查看修改代码。

2.本地拉取远程的一个新分支

只需单击同步里面的 提取。

只需要在上图中的同步中点击提取即可

常见名词解释

拉取（Pull）：将远程版本库合并到本地版本库，相当于（Fetch+Merge）

获取/提取（Fetch）：从远程版本库获得最新版本

合并（Merge）：将两个版本库进行合并操作

提交（Commit）：将所做的更改，存入本地暂存库

推送（Push）：将所做的更改，存入远程版本库

同步（Pull+Push）：先将远程库拉取，再将本地库推送到远程库，相当于（Pull+Push）

变基到（Switch）：切换分支（双击即可切换）

【更改】

​ 负责文件的修改、添加、忽略文件、删除文件提交到本地服务器，主要是：提交

【分支】

​ 负责合并分支，切换分支，主要是合并、变基到、解决冲突

【同步】

​ 远程Git存储库与本地Git存储库操作，主要是：推送、同步、拉取、获取

【设置】

​ 设置用户名、密码、远程地址、对比工具等

git pull 从远程拉取最新版本 到本地 自动合并 merge git pull origin master。。即：git pull = git fetch + git merge，这样可能会产生冲突，需要手动解决。

git fetch 从远程获取最新版本 到本地 不会自动合并 merge git fetch origin master git log -p master ../origin/master git merge orgin/master

实际使用中 使用git fetch 更安全 在merge之前可以看清楚 更新情况 再决定是否合并

初建项目并push到github上

https://github.com/swjtulwy/GitTest.git

在远程Github等服务器新建一个空Repositories，这里起名GitTest。

打开VS，新建项目GitTestProject，

右键单击解决方案，选择“将解决方案添加到源代码管理器”，

点击团队资源管理器中的同步，推送到远程仓库

此时，会显示推送成功，去github上一看

也已经有了刚创建的解决方案了。

如果在github上点击添加一个reame文件，在vs中可以拉取进行修改再提交。

提交代码

暂存想提交的文件，不想提交的右键暂存。提交临时文件，拉取再推送。

有冲突时，可以点击冲突，然后对比文件，选择使用远程的代码还是本地代码，然后再次提交。

当添加了部分代码后，如上图所示，可以马上全部提交(add+commit).此时填好commit信息就行了，在点击全部提交即可。如果不想马上提交，可以点击更改数右边的加号全部暂存(stage)，或者右键部分文件暂存。此时暂存的文件会如下图所示：

更改文件出现在已暂存的列表中，这是可以继续写代码，当写完想要与其他暂存一起提交，即可点击提交(commit)临时数据或者下拉框的其他选项。

如何把提交的代码推送到远程呢，点击同步中的推送即可

以后的日常操作，就是这些，修改后，暂存(add)，提交(commit)，推送(push)。有时候需要拉取(pull)。

到这里，关于如何在VS中简单使用git就介绍完毕，当然，还有很多强大的功能，等待大家去探索，本文也没有这么多精力去介绍。
